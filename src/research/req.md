# Требования и утверждения к MCP‑Nest

## Формальные требования
1. Система должна поддерживать транспорт **HTTP+SSE** для взаимодействия с клиентами через SSE и POST‑запросы.
2. Система должна поддерживать транспорт **Streamable HTTP** для выполнения команд и получения потокового отклика.
3. Система должна поддерживать **STDIO** транспорт для взаимодействия через стандартный ввод/вывод. 
4. MCP‑Nest должен автоматически обнаруживать и регистрировать инструменты, ресурсы и подсказки, помеченные соответствующими декораторами.
5. Валидация параметров вызова инструментов должна выполняться с помощью **Zod**. 
6. Система должна отправлять уведомления о прогрессе выполнения долгих операций (progress notifications).
7. Сервер должен позволять защиту MCP‑эндпоинтов через guard‑базированную аутентификацию.
8. Конфигурация MCP‑сервера должна задаваться через метод `McpModule.forRoot()` с указанием имени сервера и версии.
9. Система должна позволять определять динамические ресурсы с параметризуемыми URI (**Resource Templates**). 
10. Система должна обеспечивать доступ к HTTP‑контексту (например, заголовкам запроса) внутри MCP‑обработчиков.
11. Сервер должен иметь графический интерфейс пользователя, написанный на React.
12. Система должна отправлять email‑уведомления при каждом вызове инструмента.
13. Сервер должен быть эффективным
14. MCP‑Nest должен быть безопасным.
15. Сервер должен интегрироваться с внешним CRM‑решением.
16. Все инструменты MCP должны реализовывать машинное обучение внутри сервера.
