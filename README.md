### Анализатор семантического покрытия

PoC системы обнаружения покрытия требований автотестами.

#### Предварительные требования
Необходима локально развернутая ollama с набором моделей:
- rnj-1:8b-cloud
- embeddinggemma
- qwen3-next:80b-cloud

Кроме того, в качестве БД используется QDrant (настройки по умолчанию - http://localhost:6333)

#### Запуск и использование

В рамках PoC было реализовано 3 основных REST-эндпоинта:
- POST `/repo/ingest` (сбор данных о репозитории)
- POST `/req/list` (сбор требований для конкретного проекта/репозитория)
- GET `/search/calc` (получение строки с отчетом по собранным требованиям)

Создаваемый проект имеет название repo_${commitHash} (выводится в консоли по окончании сбора файлов)

Процесс получения отчета заключается в последовательном вызове этих 3 эндпоинтов. Рекомендуется последовательное 
выполнение, результат при параллельном не гарантирован
